# 正答キー バッチインポート用 仕様書（AI向け）

あなたはこれから、試験の問題冊子・正答一覧・配点表など（教員から渡される仕様書や要望）をもとに、
この試験（1バッチぶん）の**正答キー**を `answer_key` というJSON配列の形式で出力する作業を行います。

このJSONは人間が手で書くものではなく、あなたのようなチャット型AIが、渡された資料と教員とのやりとりから
作成し、教員がアプリの画面に貼り付けてインポートするためのものです。

## やること

1. 教員から、問題冊子・正答一覧・配点表（写真・PDF・テキストなど）や口頭の指示を受け取る
2. 実際にその試験で使われた設問についてのみ、`answer_key` 配列の要素を1つずつ作る
   （この解答用紙は最大80問ぶんのマーク欄を持つが、**試験によっては一部の設問しか使わない**。
   使っていない設問番号については要素を作らなくてよい）
3. 正答が資料からまだ読み取れない・教員の確認待ちの設問があれば、`correct_option` を省略したまま
   他のフィールド（配点や大問番号など）だけを埋めた要素にしてもよい（後述）
4. 最終的に `{"answer_key": [...]}` という1つのJSONオブジェクトを出力する
   （前後に説明文をつけず、JSONのみ、または明確にコードブロックで囲って出力すること）

## 出力フォーマット

各要素のフィールドは以下の通り。解答用紙の選択肢は①〜⑤（5択）。

| フィールド | 必須 | 説明 |
|---|---|---|
| `question_number` | **必須** | 設問番号（1〜80の整数）。解答用紙の実際の問題番号と一致させること |
| `correct_option` | 任意 | 正答の選択肢番号（1〜5の整数。①=1, ②=2, ③=3, ④=4, ⑤=5）。**まだ正答が確定していない設問は省略してよい**（配点や大問番号だけ先に登録しておき、後から教員が画面上で正答だけ埋める運用も可能） |
| `points` | 任意（既定1点） | この設問の配点。省略した場合、アプリ側では1点として扱われる |
| `group_number` | 任意 | この設問が属する大問の番号（整数）。大問ごとの集計をしたい場合に入れる |
| `explanation` | 任意 | 解説文。あとで成績を返却する際に学生に見せる想定のもの |
| `memo` | 任意 | 採点者向けのメモ（学生には見せない想定の備考） |
| `_comment` | 任意 | 読み取りに自信が持てない箇所、判断に迷った点などを自由記述。データベースには保存されず、教員が確認する際の参考情報としてのみ使われる |
| `_confidence` | 任意（既定`1.0`） | 0.0〜1.0の数値。正答表がはっきり印刷されていて迷いなく読めた場合は1.0、手書き・不鮮明な資料で自信が持てない場合はそれに応じて下げる（目安: 0.8未満だと確認画面で警告表示される） |

## 注意事項

- `question_number` は同じ試験内で重複してはいけない（1設問につき1要素）
- このタスクでは既存の正答キーは渡されない。上書きするか新規登録するかはアプリ側が
  自動判定するので、あなたは気にしなくてよい（同じ設問番号を再度インポートすると、
  その設問の内容は新しい値で置き換えられる）
- `points` は小数（例: 2.5点）でもよい
- 出力は妥当なJSON（末尾カンマなし、ダブルクォート使用、コメントなし）にすること
- `id` や `batch_id` のようなデータベース内部のIDは、あなたが知る・指定する必要は一切ない

## 出力例

```json
{
  "answer_key": [
    {
      "question_number": 1,
      "correct_option": 3,
      "points": 2,
      "group_number": 1,
      "explanation": "光合成の過程で放出される気体はどれか、という設問。③酸素が正答。",
      "_confidence": 1.0
    },
    {
      "question_number": 2,
      "group_number": 1,
      "points": 2,
      "_comment": "正答表のこの箇所が不鮮明で読み取れなかったため、正答は未入力にしてある。教員に確認が必要。",
      "_confidence": 0.4
    }
  ]
}
```
