# autograder

マークシート式の解答用紙をスキャンして、学籍番号・解答を自動読み取り・確認・CSV出力するための教員向けツール。

採点データという性質上、**各教員のPC上でスタンドアロンで動作**します（外部サーバーには一切送信されません）。

---

## 必要なもの

- Docker が動く環境
  - **Windows**: WSL2 を導入し、その中に Docker Engine を直接インストール（`curl -fsSL https://get.docker.com | sh`）。詳しい手順は [`docs/windows-setup.md`](docs/windows-setup.md) を参照
  - **Mac**: [Colima](https://github.com/abiosoft/colima) を利用（`brew install colima docker docker-compose` → `colima start`）。詳しい手順は [`docs/mac-setup.md`](docs/mac-setup.md) を参照
- スキャナ（ScanSnap 等、自動送り対応のもの。片面読み取りで十分）

---

## セットアップ

1. このフォルダをまるごと自分のPCに置く
2. ターミナルでこのフォルダに移動し、次を実行

   ```bash
   sh start.sh
   ```

   初回は `.env` が自動生成されます。しばらくするとブラウザが自動で開きます（開かない場合は `http://localhost:8000` を手動で開いてください）。

## 使い方

左のサイドバーに機能が2つあります。

- **答案読み取り・採点**: スキャンしたPDFのアップロード・読み取り結果の確認・採点
- **解答用紙編集・ダウンロード**: 配布用の解答用紙PDFの作成

### 1. 解答用紙を作成・ダウンロードする

サイドバーの「解答用紙編集・ダウンロード」から、試験名・実施日・担当教員名を入力して「作成」を押すと、その場で解答用紙PDFが生成されます。すべて任意項目で、未入力のまま作成すれば汎用の解答用紙になります。

「PDFダウンロード」から取得したPDFを学生数ぶん印刷して配布します（1人1枚）。

- **学籍番号（10桁）**: 上段に数字を手書きし、その下のマークシート欄も塗りつぶす
- **氏名**: 手書きのみ
- **解答**: 各設問の①〜⑤を塗りつぶす（5択×80問）

作成した解答用紙は一覧に残るので、後から試験名などを編集したり、再度ダウンロードしたりできます。

### 2. スキャンする

ScanSnap 等で、学生の答案をまとめて**1つのPDF**にスキャンします（1ページ＝1人）。300dpi・片面で十分です。

### 3. アップロードする

サイドバーの「答案読み取り・採点」を開き、PDFを選び、「備忘」欄に授業名・実施年度などをメモしてからアップロードします（あとで見返すときに分かりやすくするためのものです）。

### 4. 結果を確認・修正する

一覧画面で、学生ごとに氏名（画像）・学籍番号（画像＋読み取り結果）・解答の要約を確認できます。

実際に出題したのがQ1〜80問の一部だけの場合は、画面上部の「有効問題番号」でQいくつまで使ったかを設定してください。それより後の設問は解答選択・採点・CSV出力の対象外になります（未設定なら全80問が対象です）。

- **学籍番号**が誤読・未読み取りの場合は、その場でテキストを書き換えて保存できます
- 行をクリックすると、その学生の解答詳細ページに移動します（詳細ページ内の「前の受験生」「次の受験生」で他の学生にも移動できます）
  - 各設問について、**実際にスキャンされたマーク欄の画像**と、採用される解答が並んで表示されます。解答はどの設問でもプルダウンでいつでも選び直せます
  - **黄色い設問**は、2つ以上のマークが検出された、または薄い・部分的なマークで自動判定に自信が持てなかったものです。画像を見て正しい解答（または「無回答」）を選び直してください
  - 画像を確認した結果「無回答／無効のままで正しい」と判断した場合は、選び直す必要はなく「確認済みにする」ボタンを押せば、その設問を確認済みとして記録できます（そのまま放置すると、一覧画面の「要確認」件数にずっとカウントされ続けます）

### 5. 正答・配点を入力して採点する（任意）

正答を入力しなくても、上記までの「読み取り・確認・CSV出力」は問題なく使えます。採点まで行いたい場合は、答案一覧画面の「正答・配点」タブから設問ごとに正答（①〜⑤）・配点・大問番号・解説・メモを入力します。

- 有効問題番号の範囲内でも、正答を入力しなかった設問は採点対象から除外されます。実際に使った設問だけ入力すれば十分です
- 配点を省略した設問は1点として扱われます
- 入力するとその場で自動保存されます（画面下部の「保存済み」表示で確認できます）

**設問数が多い場合はAIインポートが便利です。** 「AIインポート」タブを開き、問題冊子や正答一覧・配点表の画像・PDFと、「AIインポート仕様書」（別タブで開きます）をチャット型AI（Claude等）に渡すと、貼り付け用のJSONを作成してもらえます。JSONを貼り付けて「プレビュー」を押すと、内容を確認・修正してから反映するかどうかを1件ずつ選べます。

正答が1問でも入力されると、答案一覧・解答詳細の各画面に得点（合計点・正誤）が表示されるようになります。

### 6. CSVをダウンロードする

答案一覧画面の「CSVダウンロード」から、次の3形式でCSVを取得できます。

- **tidy形式**: 学籍番号・設問番号・回答（数値1〜5）の3列（正答が1問でも設定されていれば得点列も追加されます）。常に利用できます
- **合計得点形式**: 学籍番号・合計得点の2列。有効範囲の全設問に正答が設定されると利用できます
- **大問別得点形式**: 学籍番号・大問ごとの得点・合計得点。有効範囲の全設問に正答・大問番号が設定されると利用できます

tidy形式のCSVは Google スプレッドシートのQUERY関数などで「学籍番号×設問番号」の横持ち表に展開できます。

### 7. 不要になったバッチを削除する

答案一覧画面いちばん下の「危険な操作」から、そのアップロード（学生の読み取り結果・画像・解答をすべて含む）を削除できます。元に戻せないので注意してください。

## アプリを止める・完全に削除する

- **一時停止**（次回すぐ再開できる。データもそのまま）: `sh stop.sh`
  - `docker compose down` を実行し、コンテナだけを停止・削除します。ビルド済みのDockerイメージと `./data` の中身（採点データ）はそのまま残るため、次に `sh start.sh` を実行すればすぐに（再ビルドなしで）再開できます。
- **完全アンインストール**（コンテナ・Dockerイメージを削除。採点データの削除は確認つき）: `sh uninstall.sh`
  - `docker compose down --rmi all` でコンテナに加えてビルド済みDockerイメージも削除し、ディスク容量を解放します。そのあと `./data` の中身（採点データ）を削除するかどうか確認されます（削除すると元に戻せません）。

### stop しないとどうなるか

`docker-compose.yml` で `restart: unless-stopped` を指定しているため、`sh stop.sh`（または `sh uninstall.sh`）で明示的に止めない限り、コンテナはPCを再起動してもDocker側で自動的に再起動し続けます。ポートは `127.0.0.1:8000` にバインドされ外部（LAN等）には公開されませんが、使わない間もメモリ・CPUを使用し続けることになるので、しばらく使わない場合は `sh stop.sh` で止めておくことをおすすめします。
