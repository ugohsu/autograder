# 今後の実装予定

現在動いているのは「解答用紙の作成・ダウンロード、マークシート（学籍番号＋5択80問）の読み取り・確認・CSV出力・採点（正答照合）」まで。
以下は未実装の拡張候補。

---

## 採点機能・追加検討事項（優先度: 中）

正答キー入力（画面から1問ずつ／チャット型AIが作成したJSONのバッチインポートの両方に対応）と、
それに基づく得点表示（答案一覧・解答詳細）は実装済み（`answer_keys` テーブル、`blueprints/answer_key.py`）。
正答が1問も設定されていないバッチでは、従来どおり採点なしで読み取り確認・CSV出力のみ行える。

残っている拡張候補:

- 項目分析（正答率・識別力）: `pandas` での集計、`matplotlib` でのヒストグラム等を想定
- 大問（`group_number`）単位の小計表示
- 得点そのもののCSV/スプレッドシート出力（現状は画面表示のみで、CSV出力は引き続き回答の生データが対象）

## 帳票出力（優先度: 中）

個人成績票PDF・成績一覧・得点分布グラフなどの出力。

- 解答用紙の試験名・実施日・担当教員名の埋め込み（`blueprints/answersheet.py`）で実証済みの通り、
  TeX Live を使わず PyMuPDF（既存の依存パッケージ）でテキストをPDFに直接描画/上書きすれば、
  Dockerイメージを重くせずに数値データ入りPDFを生成できる。個人成績票・成績一覧もこの延長で
  実装できる見込み（`answersheet/sheet.pdf` のような固定レイアウトの上に文字を重ねる方式）
- 得点分布グラフ等、画像描画が必要なものは別途 `matplotlib` の導入を検討

## 自由記述・数値解答のOCR（優先度: 低・現行の解答用紙では不要）

現在の解答用紙はマークシートのみの設計のため不要と判断済み。将来、自由記述欄を持つ解答用紙を別途設計する場合に備えてのメモ:

- `pytesseract` / `PaddleOCR`（手書き数字に強い）/ `google-cloud-vision` などが候補
- 手書き記述の意味的な自動採点（LLM judge）は `anthropic` SDK の利用を想定

## その他検討事項

- 学籍番号マーク欄も、解答と同様に複数マーク検出時の「要確認」フラグを出す仕組みを入れるか（現状は解答欄のみ対応）
- 複数バッチ・複数授業をまたいだ集計・検索
